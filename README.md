
# 🌳 אילן יוחסין: משפחת ליבנת-זיידמן (Livnat-Zaidman Family Tree)

אפליקציית ווב אינטראקטיבית ודו-לשונית (עברית/אנגלית) לחקר, ניהול והצגה של היסטוריה משפחתית מסועפת. האפליקציה ממפה למעלה מ-2,600 דמויות ומאות שנות היסטוריה (מהמאה ה-16 ועד ימינו), תוך שילוב ויזואליזציות מתקדמות, מחקר גנאלוגי, גנטיקה (DNA) וסיפורים היסטוריים.

![React](https://img.shields.io/badge/React-19-blue) ![TypeScript](https://img.shields.io/badge/TypeScript-Ready-blue) ![Vite](https://img.shields.io/badge/Vite-6-purple) ![TailwindCSS](https://img.shields.io/badge/Tailwind-RTL-cyan)

## ✨ תכונות מרכזיות

*   🌳 **תצוגת עץ אינטראקטיבית (Tree View):** 
    *   ויזואליזציה דינמית מבוססת ReactFlow הכוללת גרירה, זום וניווט חלק.
    *   פסי דורות (Generation Bands) וסידור אוטומטי של צמתים.
    *   צמתים צבעוניים המקודדים לפי מין, דור, וסמלי סטטוס היסטוריים/גנטיים (🧬 אימות DNA, 🕯️ נספי שואה, 🎗️ ניצולים, 🎖️ פרטיזנים).
    *   תצוגת תת-עץ: לחיצה על אדם מבודדת ומציגה רק את הענף הישיר שלו.
*   🗺️ **מפת נדודים עולמית (Map View):**
    *   מפה אינטראקטיבית (Leaflet) המציגה את מסלולי ההגירה של המשפחה.
    *   שימוש בקואורדינטות מדויקות (GPS) או מיקומים משוערים ממקום הלידה/פטירה.
    *   קיבוץ אשכולות (Clustering) לפי מיקום, עם חלונות קופצים (Popups) המציגים את כל ילידי המקום.
*   📅 **ציר זמן היסטורי (Timeline):**
    *   גרף עמודות כרונולוגי המציג אירועי חיים (לידות ופטירות) לפי עשורים.
    *   לחיצה על עשור מסוים מסננת ומציגה את כל הדמויות מאותה התקופה.
*   🔍 **חיפוש חכם ואבולוציית שמות (Smart Search):**
    *   מנוע חיפוש Fuzzy (באמצעות Fuse.js) התומך בעברית ואנגלית.
    *   יכולת "איחוד שמות" (Entity Resolution) - מציאת דמויות גם לאחר שינויי שם היסטוריים או הגייתיים (למשל: Castro/Kastrel, Alperovich/Halperin).
    *   סינון מתקדם לפי: טווח דורות, שם משפחה, מין, אימות גנטי (DNA) ו"קפיצות" מרחק הקרבה ליעל.
*   📚 **ארכיון נרטיבי ותובנות (Archive & Insights):**
    *   פאנל פרטים ביוגרפי עם ניווט בין קרובי משפחה.
    *   תצוגת סטטיסטיקות על התפלגות ענפים, דמוגרפיה ותפוצה גיאוגרפית.

## 🛠️ טכנולוגיות

*   **ממשק משתמש:** React 19 + TypeScript
*   **בנייה ופיתוח:** Vite 6
*   **ויזואליזציית עץ:** ReactFlow, אלגוריתם סידור Dagre

*   **בדיקות וטסטים:** Vitest (ליחידות), Playwright (ל-E2E)

## 🚀 התקנה והרצה (סביבת פיתוח)

כדי להריץ את הפרויקט באופן מקומי, יש לשבט את המאגר ולייבא את קבצי הנתונים הפרטיים שלא נמצאים בגיטהאב מטעמי פרטיות.

**1. שיבוט הריפו:**
```bash
git clone https://github.com/Flyingy37/yael_family_tree_github.git
cd yael_family_tree_github
```

**2. הוספת קבצי נתונים (חובה):**
מטעמי פרטיות, בסיס הנתונים הגולמי אינו שמור במאגר הציבורי. חובה להעתיק את קבצי המחקר מהמקור הפרטי שלכם אל תוך תיקיית `data/`:
*   `data/canonical.csv`
*   `data/curated.csv`
*(הערה: לעיון במבנה הנתונים הנדרש, ראו את הקובץ `data/data_dictionary.md` במאגר).*

**3. התקנת תלויות:**
```bash
npm install
```

**4. הרצה בפיתוח:**
הפקודה הבאה תקמפל את קבצי ה-CSV לקובץ ה-JSON הנדרש (דרך הסקריפט `build-graph.ts`) ותפעיל את שרת הפיתוח:
```bash
npm run dev
```
האפליקציה תיפתח בדפדפן בכתובת: `http://localhost:5173`

## 🗺️ נתיבים עיקריים (Routing)

האפליקציה תומכת בניווט דו-לשוני (he/en).

| תיאור | כתובת (URL) |
| :--- | :--- |
| **דף הבית** | `/` |
| **חוקר העץ** (מעבר בין עץ / מפה / ציר זמן דרך טאבים) | `/he/tree` או `/en/tree` <br>*(פרמטרים: `?view=map, timeline, stats`)* |
| **תובנות וסטטיסטיקה** | `/he/insights` או `/en/insights` |
| **ארכיון נרטיבי** (סיפורים) | `/he/archive` או `/en/archive` |
| **כרטיס אדם אישי** | `/he/person/@I123@` *(לפי מזהה ה-ID בגרף)* |

*(הערה: נתיבי `/explore/...` ישנים מפנים אוטומטית ל־`/he/tree` או ל־`/he/insights`).*

## 🔒 מדיניות פרטיות ואבטחת נתונים (אזהרה חמורה)

קלט הבנייה של האפליקציה מתבסס על הקבצים `canonical.csv` ו-`curated.csv`. קבצים אלו מכילים מידע גנאלוגי, גנטי ואישי רגיש. 
**חל איסור מוחלט לדחוף (Push) נתונים רגישים למאגר הציבורי ב-GitHub.**

**אין לפרסם בריפו הציבורי את הנתונים הבאים:**
*   נתוני DNA גולמיים (Raw DNA), רשימות התאמות (Matches), ערכי cM או Haplogroups.
*   ייצואי Chromosome Browser או מזהי ערכות מעבדה.
*   קבצים המכילים נתונים מזהים של **קטינים** או **אנשים חיים** (שם מלא + קשרי משפחה + תאריכי/מקומות לידה מדויקים).
*   דוחות מחקר פנימיים (מתיקיית `reports/`).

**פעולות במקרה של חשיפת מידע:**
*   ריפו ציבורי אפשרי **רק** עם נתונים מסוננים/מצונזרים, או לאחר ניקוי עמוק של היסטוריית ה-Git. 
*   אם חומר רגיש דלף בעבר לענף ה-`main` הציבורי: חובה להפוך מיד את הריפו ל-**Private**, להסיר את הקבצים ממעקב Git (`git rm --cached`), ולבצע שכתוב היסטוריה (History rewrite) באמצעות כלי כמו `git filter-repo` בטרם החזרת המאגר לסטטוס Public. במקרים חמורים, מומלץ ליצור Repository חדש ונקי.
