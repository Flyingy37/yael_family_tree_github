# 🌳 אילן יוחסין - משפחת ליבנת זיידמן

אפליקציית ווב אינטראקטיבית לצפייה באילן היוחסין של משפחת ליבנת-זיידמן.
גודל העץ נקבע אך ורק מקבצי הנתונים המקומיים (`data/canonical.csv`, `data/curated.csv`) — **אינם נכללים בריפו ציבורי** מסיבות פרטיות (ראו למטה). בגרסה מלאה מקומית ייתכנו **אלפי אנשים** (למשל 2,700+); אחרי סקריפט הבנייה ומיזוג כפילויות המספר הסופי משתנה לפי הנתונים.

## תכונות

### 🌳 תצוגת עץ
- ויזואליזציה אינטראקטיבית עם גרירה, זום וניווט
- צמתים צבעוניים לפי מין ודור
- קישורים בין הורים, ילדים ובני זוג
- **תצוגת עץ משנה** — לחיצה על אדם מציגה רק את העץ שלו

### 🗺️ מפת עולם
- מפה אינטראקטיבית — מוצגים **רק** אנשים שיש להם קואורדינטות בנתונים (המספר תלוי בקובץ המקור, לא קבוע)
- אשכולות לפי מיקום עם popup שמציג את כל האנשים באותו אזור
- צבעים לפי מין

### 📅 ציר זמן
- גרף עמודות לפי עשורים (לידות ופטירות)
- לחיצה על עשור מציגה את כל האירועים בתקופה

### 🔍 חיפוש וסינון
- חיפוש fuzzy בעברית ואנגלית (שמות, מקומות, קשר ליעל)
- סינון לפי: טווח דורות, שם משפחה, מין, מידע גנטי, קפיצות מיעל
- פאנל פרטים עם ניווט בין בני משפחה

## התקנה

```bash
# שיבוט הריפו
git clone https://github.com/Flyingy37/yael_family_tree_github.git
cd yael_family_tree_github

# קבצי נתונים (לא בריפו ציבורי): להעתיק ל־data/ ממקור פרטי
#   data/canonical.csv
#   data/curated.csv
# ראו data/data_dictionary.md

# התקנת תלויות
npm install

# הרצה בפיתוח (בונה את הנתונים ומפעיל שרת)
npm run dev
```

האפליקציה תיפתח ב- http://localhost:5173

## טכנולוגיות

- **React 19** + **TypeScript** — ממשק משתמש
- **Vite 6** — בנייה ופיתוח
- **ReactFlow** — ויזואליזציית עץ אינטראקטיבית
- **Leaflet** — מפה אינטראקטיבית
- **Dagre** — אלגוריתם layout לעץ
- **Fuse.js** — חיפוש fuzzy
- **Tailwind CSS** — עיצוב RTL

## נתונים

קלט הבנייה הוא `data/canonical.csv` + `data/curated.csv` (פורמט מתועד ב־`data/data_dictionary.md`).  
הפלט הוא `public/family-graph.json`, והאפליקציה טוענת אותו בזמן ריצה. **בשיבוט ציבורי בלי קבצי CSV** הבנייה לא תצליח או שהגרף יהיה ריק — זה צפוי; יש להעתיק את הנתונים המקומיים ל־`data/` ואז להריץ `npm run dev` או `npm run build`.

בגרסה מלאה ייתכנו גם שדות הערות עם **DNA** — לכן הקבצים האלה **לא נדחפים** לריפו ציבורי.

## פרטיות ופרסום

- **לא לפרסם בריפו ציבורי:** raw DNA, התאמות, cM, haplogroups, ייצואי chromosome browser, רשימות matches, מזהי ערכות, קבצי קטינים עם שם מלא + קשר משפחה + תאריך/מקום, או דוחות פנימיים (`reports/`).
- **ריפו ציבורי אפשרי רק** אחרי ניקוי היסטוריית Git מתוכן רגיש (או ריפו חדש) + שימוש בנתונים **מסוננים** או החזקת הריפו כ־**Private**.
- אם חומר רגיש כבר היה ב־`main` ציבורי: להפוך את הריפו ל־**Private**, להסיר מהמעקב, ולבצע **history rewrite** (`git filter-repo` וכו׳) לפני החזרה ל־Public.
